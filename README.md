# Project SENTINEL: The Autonomous Wealth Governance Lifeform

> "To the Strategic Singularity and Beyond."

## 1. Vision: Why SENTINEL?
Project SENTINELは、個人の資産を自律的に統治・防衛するために設計された「外部脳」のプロトタイプです。
既存の金融システムが抱える情報の断片化とインターフェースの制約を打破し、自身の資産形成ロードマップに基づき、意志をダイレクトに数字へ同期・最適化することを目的としています。

## 2. The 6-Layer Architecture (DB Structure)
本システムは、以下の6つの独立した論理データベース（DB）によって制御されています：
1. **Core DB**: 目的と野心の定義。
2. **Financial DB**: 4重バックアップで保護された資産データ基盤。
3. **Compliance DB**: システムの倫理および法的境界線の定義。
4. **History & Philosophy DB**: 金融史と投資哲学に基づく意志決定支援。
5. **Defense DB**: 外部リスクおよび資産棄損を遮断するプロトコル。
6. **Agent DB**: 自律的に思考・監視・行動するエージェントの核。

## 3. Current Status: Phase 1 (Sensory Organ)
現在は「感覚受容器（Sensory Organ）」の構築フェーズです。
特定の金融サイトのDOM構造を解析し、以下の資産クラスをリアルタイムで検知・統合監視する機能を実装しています。

- **Physical Assets**: Gold, Silver, and Platinum (Real-time tracking enabled)
- **Crypto Assets**: Bitcoin (Strategic accumulation monitoring)

## 4. Technical Features
- **Dynamic DOM Scraping**: `createTreeWalker` 等を用いた、動的要素からの堅牢なデータ抽出。
- **Auto-scaling Overlay UI**: 保有銘柄の出現状況に応じた動的なモニタリングバッジの生成。
- **Data Integrity**: ミラーリング設定によるデータ消失防止ロジックの搭載。
